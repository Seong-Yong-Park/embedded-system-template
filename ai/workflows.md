# AI Workflows

## Overview
This document describes how to use AI tools in this project and various scenarios.

## Usage Scenarios

### Code Analysis
- **When**: Understanding new codebase, pre-refactoring analysis
- **How**: Use code analysis prompts from `ai/prompts/`

### Code Review
- **When**: Before PR, reviewing major changes
- **How**: Apply code review rulesets from `ai/rulesets/`

### Documentation
- **When**: Writing design documents, interface specifications
- **How**: Use documentation prompts from `ai/prompts/`

### Root Cause Analysis (RCA)
- **When**: Bug analysis, system failure analysis
- **How**: Apply RCA rulesets from `ai/rulesets/`

### Test Case Generation
- **When**: After developing new features
- **How**: Generate test cases based on requirements

## Prompt Management
- Store frequently used prompts in `ai/prompts/`
- Version control project-specific prompts

## Ruleset Management
- Store code style, review criteria, etc. in `ai/rulesets/`
- Apply team standards consistently
